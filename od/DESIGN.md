# WaveRider Core — Quantum Terminal

> OD mirror of `@waverider/design-system`. **Code is the source of truth**
> (`tokens/core.css`); this file is generated context for design work.
> When core.css changes, re-propagate — OD has no inheritance.

## 1. Visual Theme & Atmosphere

Not a SaaS dashboard. A trader's terminal.

Bloomberg and Think-or-Swim gave the shape: deep background, high density,
numbers first, every pixel serving a decision. Those tools are a 1990s design
language — we reinterpret the same purpose with 2020s execution quality.

Three judgment rules that outrank every specific spec:

1. **Numeric clarity always wins.** When "looks good" fights "the number is
   legible," legibility wins. Contrast, size and padding all serve reading speed.
2. **Density like a terminal, not a marketing page.** Traders don't need
   whitespace to breathe; they need maximum signal per glance.
3. **Consistency beats brilliance.** Every page beautiful but different reads
   instantly as AI-generated. Consistency lets attention land on the data.

## 2. Color

Dark is the default polarity; light exists for marketing surfaces.

**Surfaces** — a 5-step ramp. +1 step = floats one notch above its parent.
Never skip a step. `--wr-surface-void` `#05080D` → `base` `#0A0F1A` →
`raised` `#111827` → `elevated` `#1F2937` → `overlay` `#374151`.

**Accent** — cyan `#06B6D4`. Signals technical precision. Used for calls to
action and focus only. Never decorative.

**Direction** — named `up`/`down`, never green/red, so a colorblind-safe or
locale-inverted palette is a value swap. Up `#22C55E`/`#4ADE80`/`#86EFAC`,
flat `#94A3B8`, down `#FCA5A5`/`#F87171`/`#EF4444`.

**Charts** — an 8-hue categorical series that deliberately excludes pure green
and red, so series color never collides with directional meaning.

**Hard rule:** color is never the only channel. Every directional value ships
a redundant ▲/▼ glyph. ~8% of men cannot separate these hues and this audience
skews heavily male.

## 3. Typography

| Role | Face | Why |
|---|---|---|
| Display | Outfit | Headings, hero numbers. Has personality. |
| Body | Plus Jakarta Sans | Prose, labels. High legibility. |
| Mono | JetBrains Mono | All trading numerals. Non-negotiable. |

Two scales, on purpose: display (48→22px) is narrative, data (24→12px) runs
tighter line-height to survive dense tables.

All numerals are mono **and** tabular (`tabular-nums`). Without tabular figures
prices jitter horizontally on every tick and the eye cannot track a column.

**Inter is banned** — it is the homogenizing signature of AI-generated apps.

## 4. Spacing

8px base rhythm, 4px half-step. Every gap is a multiple.

Density is a token, not a rewrite: `compact` (28px rows, scanners),
`standard` (36px), `comfortable` (44px, journaling and marketing). Row height,
cell padding and table type size all move from one attribute.

## 5. Layout & Composition

**The 10-second rule.** Every page answers exactly one question. Make the
answer the largest, brightest, most prominent element; demote everything else.
If it takes more than 10 seconds to answer, the composition failed — not the
data.

Three levers, and only three: **size**, **color**, **position**. One screen,
one focal point. If two elements compete for dominance, the page's single
question hasn't been decided yet.

**Decision pyramid** (ticker drawer): the four things that drive an entry —
RS, Vol×, Catalyst, Setup — go top, largest. Everything else is secondary.

## 6. Components

Anatomy is shared; only accent, density and one signature vary per app.

Radius: functional elements ≤8px, containers ≤12px, modals 16px. Terminal feel
needs sharper corners than SaaS.

Search is a primary action, not a utility: visible field at the top-left sight
line with a ⌘K hint. Never an icon-only affordance in a corner.

Focus must be unmistakable — these are keyboard-driven surfaces. 2px accent
ring, 2px offset. Hit targets ≥24px on dense surfaces, ≥44px anywhere touchable.

## 7. Motion & Interaction

150–300ms. Longer reads as slow, and traders won't forgive it.

Motion is only ever feedback ("this datum just changed") or a state transition.
Never decoration. Flash up/down on tick, pulse for live data, scan-line for an
active engine.

Shadow means **glow**, not a light source. Glow says "something is happening
here." Static elements get none. (Light polarity is the one exception — glow is
invisible on white, so elevation reverts to conventional shadow.)

`prefers-reduced-motion` collapses all durations to ~0 while preserving
directional meaning. A UI that flashes on every tick is unusable otherwise.

## 8. Voice & Brand

Precise, factual, unhyped. The product's authority comes from being right about
numbers, not from adjectives.

**Numbers are never generated by an LLM.** In analyst notes, the backend
computes every value as verified fact; language only assembles those facts into
prose. One wrong number destroys trust permanently.

Catalyst is a filter, not a footnote: a breakout with no reason to move is a
weaker signal and should be labeled as such.

## 9. Anti-patterns

- Inter + purple gradient + card-inside-card — the v0/Lovable house style.
- Raw Tailwind palette colors (`text-gray-400`, `text-green-500`). Reasonable-
  looking, outside the system, and how consistency erodes silently.
- Purposeless drop shadows. Shadow is glow, glow is activity.
- More than two background colors for visual separation.
- Decorative animation, or anything over 300ms.
- Trading numerals in a proportional face, or without tabular figures.
- Color as the sole carrier of direction.
- Inventing a visual language instead of consulting macOS / trading terminal
  conventions first.
- Deciding aesthetics after the code is written.
